מה נלמד?

כשכותבים תוכנית היא כוללת לרוב "בלוקים": פסקאות של מספר שורות קוד ששייכות לאותו רצף לוגי. לפני שנכיר את בלוק התנאי, נכיר את המבנה הכללי של הבלוק הפייתוני.

להלן דוגמה לבלוק בפייתון:

if (age < 18):
   print("see you next time")
   guests -= 1

מהם מאפייני הבלוק?

השורה הראשונה בבלוק היא שורת הפקודה ולאחריה יגיע רצף שורות קוד ששייך לבלוק. שורת הפקודה תמיד מסתיימת בנקודתיים (כמו פקודת התנאי, if, בדוגמה) ויתר שורות הבלוק נמצאות בהזחה לשורה הראשונה.
הזחה (אינדנטציה, indentation) היא הרחקה של שורה מהשוליים.


הזחה להמשך קריאה

ההזחה בפייתון יכולה להתבצע על ידי רווחים או על ידי טאב (TAB). בקורס זה נעשה שימוש בארבעה רווחים.




קונבנציות הזחה בבלוקים להמשך קריאה

קריאה נוספת על קונבנציות הזחה בבלוקים תוכלו למצוא בקישור הזה.




תיחום בלוקים להמשך קריאה

הביטו שוב בדוגמת הבלוק בפייתון והשוו אותה לבלוק זהה שנכתב בשפת C:

if (age < 18)
{
   printf("see you next time");
   guests -= 1
}

שימו לב שההזחה בפייתון מקבילה לסוגריים המסולסלים { } בשפות תכנות אחרות, כדוגמת: ג'אווה, C++, C ועוד.




IndentationError להמשך קריאה

מה יקרה אם נשלב הזחות מיותרות, כלומר נכניס טאב או רווחים בתוך הקוד? נסו להריץ את הדוגמה הבאה:

>>> debugging = "Let's see what happens when I use extra spaces"
>>>  print(debugging)
  File "<stdin>", line 1
   print(debugging)
   ^
IndentationError: unexpected indent

מדוע מתקבלת הודעת שגיאה בהרצת קוד זה?
מבנה הזחה שאינו נכון עלול להוביל לשגיאה מסוג: IndentationError. במקרה שלנו, הזחה בלתי צפויה.
מדוגמה זו אפשר להסיק שהזחות במקומות הנכונים אינן בגדר המלצה (במסגרת קונבנציות לכתיבה נכונה ולשיפור הקריאות), אלא הן חלק מהתחביר של פייתון. חשוב לא להוסיף רווחים מיותרים או טאבים מיותרים, ולא לשלב בין השניים.




הפקודה pass להמשך קריאה

לעיתים בכתיבת תוכנית אנחנו פותחים בלוק אבל עדיין לא "ממלאים" אותו בקוד. במקרה כזה נשתמש בפקודה שמדלגת - pass.
פקודה זו שימושית בין השאר כשרוצים להריץ חלק מתוכנית כדי לבדוק אותה, אך לפני שכתבנו חלקים אחרים.
דוגמה לשימוש ב-pass:

if python_grade > 90:
  print("I'm so proud of you!")
else:
  pass # TODO: print(<Add a motivational statement>)
